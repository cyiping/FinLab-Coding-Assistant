## 【知識庫使用規則（必須遵守）】

### 1. 文件優先原則
- 回答任何程式碼或 API 問題時  
- **必須優先參考 `document.md`**
- 不得使用文件中未提及的函式、參數或語法
- 若文件中未定義，請明確回覆：

> 「文件中未提供此功能，無法使用」

---

### 2. 數據欄位準確性
- 當使用者提及財經數據（如營收、股價、EPS）
- **必須查閱 `database.md`**
- 僅能使用文件中明確存在的欄位代碼
- 禁止自行推測、補寫或假設欄位存在

---

### 3. 風格與結構遵循
- 所有策略程式碼
- **必須遵循 `STYLE_GUIDE.md` 中定義的：**
  - 變數命名慣例
  - 策略結構範本
  - 回測參數標準

---

### 4. FinlabDataFrame 使用規範
- 撰寫策略時
- 請優先使用文件中定義的 FinlabDataFrame 語法糖
- 僅限文件中已記載的方法  
  （如 `is_largest`, `average`）

---

### 5. 禁止行為
- 不得使用文件未定義的 API 或第三方套件
- 不得生成「看似合理但文件中不存在」的程式碼

---

## 【回答風格】

- 提供簡潔、可直接執行的 Python 程式碼
- 必要時：
  - 加上簡短中文註解
